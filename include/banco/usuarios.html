MARIADB - CRIANDO USUARIO REMOTO(atualizado)

    // Acessando o mysq/mariadb via terminal
$ sudo su
$ mysql

    // criando usuario LOCALMENTE
MariaDB [(none)]> CREATE USER 'remoto'@'local' IDENTIFIED BY 'mengo';

    // criando o MESMO usuario REMOTO
MariaDB [(none)]> CREATE USER 'remoto'@'%' IDENTIFIED BY 'mengo';

    // Concedendo privilégios REMOTOS para o usuario
MariaDB [(none)]> GRANT ALL PRIVILEGES ON *.* TO 'remoto'@'%' IDENTIFIED BY 'mengo' WITH GRANT OPTION MAX_QUERIES_PER_HOUR 0 MAX_CONNECTIONS_PER_HOUR 0 MAX_UPDATES_PER_HOUR 0 MAX_USER_CONNECTIONS 0 ;

SQLSERVER - DERRUBANDO USUARIOS

Caso haja a necessidade de DERRUBAR USUARIOS por qualquer circustancia há uma funcionalidade
você pode ter que derrubar os usuarios para fazer um export ou import de determinada database por exemplo. 
USE [master]

ALTER DATABASE [SISCLAS_DEV]
SET SINGLE_USER --SETA como usuário unico para fechar conexões antes de restaurar backup

    CODIGO ANSISQL

SET MULTI_USER; --volta para multi user depois de restaurar
GO
