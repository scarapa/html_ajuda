COMANDOS

TAG COMANDO DESCRIÇÃO
RUN docker run 1 - baixar imagem(docker imagem pull)
2 - criacao de container (docker image create)
3 - execucao de container no modo interativo(docker containter start + docker container exec)
RUN docker container run -d --name nginx -p 8080:80 -v $(pwd):/usr/share/nginx/html nginx --name --> renomear a imagem baixada
-p --> portas
-v --> link apontando volume do container para maquina local
$(pwd) --> diretorio atual da onde foi feito o comando
docker run --name NOME_CONTAINER image nomear o container em modo de EXECUCAO
CONTAINER
LISTAR docker ps Listar container em execução
LISTAR docker ls -a Listar TODOS containers
LISTAR docker ps -l Listar TODOS containers
INICAR docker container start -ai -a : ter acesso ao terminal
-i modo iterativo
PARAR docker container stop -a : ter acesso ao terminal
-i modo iterativo
LOGS
docker container log container_nome
docker container inspect container_nome
IMAGENS
BAIXAR $ docker image pull IMAGEM_NOME
DELETE $ docker image rm IMAGEM_NOME
TAG $ docker image tag IMAGEM_VERSAO
CONSTRUCAO $ docker image build
PUSH $ docker image push

OBS[1]: o metodo RUN SEMPRE cria um novo container
OBS[2]: modo interativo: o container esta sendo executado ;modo daemon: o container esta sendo executado em background
