Caso 01

Foram feitas atualizações e por acidente/descuido, não foi criado um branch novo ou foi feita a modificação no BRANCH errado.
No caso abaixo foi feita modificações no BRANCH ERRADO, então precisamos por o que foi trabalhado em um BRANCH CORRETO e só então depois comitar.

1º De dentro do BRANCH ERRADO, faremos esse comando para jogarmos o que foi trabalhado em um local temporario; no git esse local temporario é uma pilha

$ git stash
Verificar se foi pra pilha, se habitue a usar bastante esse comando. 
$ git status

2º Voltar pro BRANCH MASTER

$ git checkout master

3º Criar um branch novo, para receber as modificacoes jogadas na pilha

$ git checkout -B nome_branch_novo

4º De dentro do novo branch adicionaremos o que está na pilha ao branch

$ git stash apply

5º SEMPRE trabalhar no MASTER , fazer MERGE do branch em que foi trabalhado com o MASTER

$ git merge nome_branch

6º fazer o PUSH

$ git push origin master

7º pra finalizar limparemos a pilha e deletaremos o branch

$ git stash drop 
$ git branch -D nome_branch
