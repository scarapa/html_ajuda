Criando repositório INEXISTENTE

Caso seja um repositório que você não possua

$ git clone git@git.emater.tche.br:49583/pbsm.git
Posteriormente é aconselhável criar/editar o arquivo README.md, pois o mesmo deveria ser o primeiro contato com o projeto.

Criando repositório EXISTENTE

1º entrar na pasta do projeto

$ git init

2º Definir a forma de comunicação

http ou ssh escolha uma delas, comigo funcionou a HTTP

$ git remote add origin SEU_LINK_DO_REPOSITORIO(http ou ssh)

3º Escolhido a forma de comunicação basta enviar o projeto para o repositório

$ git add .
$ git commit -m "mensagem"
$ git push -u origin master

Cores

Configurar o editor de texto padrão que será utilizado quando o Git precisar que você digite uma mensagem. Por padrão, Git usa o editor padrão do sistema, que é geralmente Vi ou Vim. Caso você queira utilizar um editor diferente, tal como o Emacs, você pode executar o seguinte: 
$ git config --global core.editor emacs
