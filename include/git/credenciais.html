Quando instalamos o GIT, ele usa um sistema de credenciais para autenticar no servidor, essas credenciais são gerenciadas por ele ou podem ser gerenciadas pelo Windows ou Linux.

USUARIO

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

TIPOS

Para se descobrir qual gerenciador de credencial seu git está usando, use o comando: 
$ git config --global credential.helper
Alguns retornos podem ser:
Padrão: o GIT não utiliza um gerenciador, as senhas são informadas a cada conexão. cache: Ele faz um cache de usuário e senha em memória por 15min, após isso ele pergunta de novo a credencial na conexão. store (antigo manager): Ele guarda o usuário/senha em um arquivo de texto, não criptografado, em disco que nunca vence, somente é atualizado quando sua senha tem que ser alterada. osxkeychain: Para usuários de MacOS, ele utiliza sistema de chaves nativo do MacOS. wincred: Estão sendo usadas as credenciais do Windows Credential Manager (Gerenciador de Credenciais do Windows) Há outros ainda, mas estas são os mais comuns.

ALTERANDO O GERENCIADOR

Depois de rodar o comando acima, para descobrir qual está sendo usado, você quer trocar pra outra, por exemplo a wincred. Para isso basta seguir o comando abaixo: 
$ git config --global credential.helper wincred

DELETANDO CREDENCIAIS

Muitas vezes caímos em problemas de termos nossas credenciais alteradas e o GIT não atualiza. Podemos então apagar as atuais, para que ele nos peça por usuário e senha novamente, armazenando assim as novas credenciais.

Com o comando abaixo, o GIT irá solicitar novamente seu usuário e senha: 
$ git config --global --unset credential.helper
Esse comando limpa como o GIT guarda sua senha.

Caso você esteja utilizando o wincred (credenciais do windows), pode ir no Gerenciador de Credenciais do Windows e apagar as credencias que começam com “git:”
