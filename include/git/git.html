.IGNORE

CRIAR arquivo .gitignore - Esse arquivo é responsável por apontar as pastas e/ou arquivos que não irão ser comitados

Exemplo do arquivo .gitignore 
/nbproject
/fpdf
/tmp
/anexos
No exemplo acima o arquivo vai ignorar essas pastas indiferente das alterações realizadas nelas.
OBS: É possível inserir o próprio arquivo .gitignore , entretanto é interessante deixar o arquivo nos commits.

BRANCH

CRIAR Branch novo

git checkout -b funcionalidade_x

RETORNAR para o Branch MASTER

git checkout master

REMOVER o Branch

-D maisculo funciona como o -force, deleta mesmo com erros.
-d minusculo se tiver erros nao deleta 
git branch -D funcionalidade_x

ALTERAR o nome do Branch 

git branch -m nome_atual nome_novo

LISTAR Branchs

git branch

ENVIAR o Branch 

git push origin

COMMIT

Adicionar TODOS os arquivos

git add .

Adicionar os arquivos INDIVIDUALMENTE

git add CAMINHO/ARQUIVO

Adiciona os arquivos por pasta no exemplo do comando é a pasta include

git add include/*

dicas COM O GIT - de verdade \o/

DICA 01

Como proceder após comitar um branch diferente do MASTER
1º Após o commit do branch 'novo' fazer checkout para o master 
git checkout master
2º pegar o utimo commit do branch nomeado. 
git cherry-pick NOME DO BRANCH
3º antes de fazer o PUSH , fazer o PULL do branch MASTER
git pull origin master
git push origin master

DICA 02

comitando alteracoes de outro branch para o ORIGIN MASTER 
git merge NOME_DO_BRANCH 
git push origin master

DESFAZENDO COISAS

resetando todo o branch(NAO RECOMENDAVEL) 
git reset --hard
