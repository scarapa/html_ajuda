CANDIDATOS

$ apt-cache policy <programa>
retorna a versao do candidato possivel para instalacao

exemplo: 
$ apt-cache policy nodejs
nodejs:
  Instalado: (nenhum)
  Candidato: 10.19.0~dfsg-3ubuntu1
  Tabela de versão:
    10.19.0~dfsg-3ubuntu1 500
    500 http://br.archive.ubuntu.com/ubuntu focal/universe amd64 Packages

INSTALACAO

padrao

$ apt install <programa>

gdebi

primeiramente baixando o pacote

instalando 'instalador'

$ apt install gdebi

download programa no formato gdebi

no exemplo abaixo é utlizado o chrome 
$ https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb -O /tmp/google-chrome-stable_current_amd64.deb

instalando

$ gdebi /tmp/google-chrome-stable_current_amd64.deb
OBSERVACAO: utilizando o parametro -y é confirmado a instalacao 
$ apt install <programa> -y

LOCALIZACAO

pasta

$ dpkg -S python2.7
  /usr/lib/python2.7/

executavel

$ type -a python2.7
  python2.7 is /usr/bin/python2.7
  python2.7 is /bin/python2.7

VARIAVEL DE AMBIENTE

$ export PATH=$PATH:/home/beto/meusprogramas
onde:
  • export comando do bash para definir uma variável acessível por qualquer processo filho dele;
  • PATH o nome da variável, sempre em LETRAS MAIÚSCULAS, que é o padrão POSIX;
  • $PATH ao usar $NOME_DA_VARIÁVEL, retorna o valor atual armazenado;
  • : dois pontos é o separador de valores convencionado para PATH;
  • /home/beto/meusprogramas o novo valor que estamos adicionando.
    • Para ver como ficou, basta consultar assim: 
    $ printenv PATH

    diolinux
    acervolima
